## Part 2 — Questionnaire Generation

Add a 'Generate questionnaire' button on the assessment detail page, below the DORA classification card.
When to Show Prominently
Show as a highlighted primary button when any of these conditions are true:
- No trust portal found AND no manual certs added
- No SOC2 or ISO27001 in certifications
- DORA classification is Important or Critical
- Going concern flag raised
- Company status not Active
- Non-UK/EU/US jurisdiction
Otherwise show as a secondary button: 'Generate due diligence questionnaire'
Questionnaire Content
Section 1 — Information security (always included):
- Q1. Do you hold a current SOC 2 Type II report? If so, please provide the audit period and issuing auditor.
  - Pre-fill if no cert found: 'Our records show no SOC 2 certification was identified via public trust portals. Please confirm current status.'
- Q2. Do you hold a current ISO 27001 certificate? If so, please provide the certificate number, issuing body, and expiry date.
- Q3. Do you have a documented Information Security Management System (ISMS)?
- Q4. When did you last conduct a penetration test? Please summarise the scope and key findings.
- Q5. Do you have a formal vulnerability disclosure or bug bounty programme?
Section 2 — Data protection (if DORA form personal data = Yes):
- Q6. Who is your nominated Data Protection Officer?
- Q7. In which countries is personal data processed and stored?
- Q8. What is your data retention policy for personal data processed on our behalf?
- Q9. Have you suffered any personal data breaches in the last 24 months? If so, please describe.
  - Pre-fill if breach score < 75: 'Our screening identified potential breach incidents. Please provide details and remediation steps taken.'
Section 3 — Operational resilience (always included):
- Q10. What is your documented Recovery Time Objective (RTO)? [Pre-fill from contract details if entered]
- Q11. What is your documented Recovery Point Objective (RPO)? [Pre-fill from contract details if entered]
- Q12. When did you last test your business continuity plan? Please summarise the outcome.
- Q13. Do you have a documented exit and transition plan that would allow us to migrate away from your service within a defined timeframe?
Section 4 — Sub-outsourcing:
- Q14. Do you sub-outsource any material components of the service you provide to us? If so, to whom and in which jurisdictions?
- Q15. How do you manage the security of your own third-party suppliers?
Section 5 — Conditional questions:
- Q16 (if going concern flag raised): Please provide an update on the financial position referenced in your most recent audit report, including any remediation steps taken.
- Q17 (if breach score < 75): Please describe the breach incident(s) identified in our screening, including remediation steps and current status.
- Q18 (if non-UK/EU/US jurisdiction): Are you subject to any laws or regulations in your jurisdiction that could conflict with your obligations to us under UK/EU data protection law?
- Q19 (if DORA Critical): Can you provide evidence of your compliance with DORA Article 28 obligations as they apply to your service?
- Q20 (if DORA Critical): Do you have a lead overseer designation or are you subject to regulatory oversight in any EU member state?
PDF Output Format
Cover page includes:
- 'THIRD-PARTY SECURITY QUESTIONNAIRE'
- Addressed to: [vendor name]
- Prepared by: [org name]
- Date: [today's date]
- Reference: Assessment ID [id]
- Instructions: 'Please complete all sections and return within 14 days. Where questions are pre-populated with information from our records, please confirm or correct as appropriate.'
Each question formatted as:
- Bold question number and text
- Italicised pre-populated context note if applicable
- 'Response:' label followed by blank lines for written answer (5–6 lines of space)
- Thin divider between questions
Footer on each page: Fides seal (small), assessment ID, 'Confidential — [org name]'
API Route
```
POST /api/assessments/[id]/questionnaire
```
- Generates and returns the PDF
- Creates/updates questionnaire row in database with status SENT and generated_at timestamp
- Returns PDF as download

## Part 3 — Cert Expiry Alerts Cron Job

vercel.json
Create vercel.json in project root (or update if exists):
```
{
  "crons": [
    {
      "path": "/api/cron/cert-alerts",
      "schedule": "0 9 * * *"
    }
  ]
}
```
Cron Route
```
Create /api/cron/cert-alerts/route.ts
```
- Secured: check request header x-vercel-cron-secret === process.env.CRON_SECRET — return 401 if mismatch
- Query all certifications where expiry_date IS NOT NULL AND deleted_at IS NULL AND expiry_date <= NOW() + INTERVAL '90 days'
- For each cert, create cert_alert rows for applicable thresholds:
  - EXPIRING_90_DAYS: expiry within 90 days
  - EXPIRING_30_DAYS: expiry within 30 days
  - EXPIRING_7_DAYS: expiry within 7 days
  - EXPIRED: expiry_date has passed
- UNIQUE constraint on cert_id + alert_type + DATE(created_at) prevents duplicate alerts on the same day
- Return { processed: N, alertsCreated: N }
Add to Vercel environment variables AND .env.local AND .env.example:
```
CRON_SECRET=<generate with crypto.randomUUID()>
```
Dashboard Cert Alerts Widget
Below recent assessments on the dashboard, show 'Certification alerts' section only when active alerts exist.
Each alert row shows:
- Cert type (human readable) and vendor name
- Alert badge colour-coded:
  - 90 days: background #FAEEDA, text #633806
  - 30 days: background #FEE2CC, text #7C3A0A
  - 7 days: background #FCEBEB, text #791F1F
  - Expired: background #3C1515, text #FCEBEB
- Expiry date
- 'View assessment →' link
